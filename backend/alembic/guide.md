🛠 Инструкция по работе с миграциями (Alembic)
Этот документ описывает процесс изменения структуры базы данных в проекте FastAPI + PostgreSQL.
Перед началом
Убедись, что:
1. Виртуальное окружение активировано.
2. Твой локальный PostgreSQL запущен.
3. Переменная DATABASE_URL в .env или alembic.ini настроена верно.

Шаг 1: Опиши изменения в моделях
Внеси изменения в свои модели SQLAlchemy (например, в файле models.py).
Например, добавь новое поле:
class Task(Base):
    __tablename__ = "tasks"
    # ... старые поля ...
    is_urgent = Column(Boolean, default=False)  # Новое поле


Шаг 2: Создай файл миграции
Выполни команду в терминале. Alembic сравнит твои модели с текущим состоянием базы и создаст скрипт.
alembic revision --autogenerate -m "add_is_urgent_to_tasks"

• -m — это краткое описание того, что ты изменил (используй английский, без пробелов, через нижнее подчеркивание).


Шаг 3: Проверь созданный файл
Перейди в папку alembic/versions/. Найди там свежий файл (его имя начинается с хеш-кода).
Обязательно посмотри глазами, что внутри функций upgrade() и downgrade() именно те изменения, которые ты планировал.
Иногда автогенерация может не заметить изменение типа данных или переименование таблицы.

Шаг 4: Примени миграцию к базе
Чтобы изменения реально отразились в твоей локальной базе Postgres, выполни:
alembic upgrade head

• head — означает «обновиться до самой последней версии».
🆘 Что делать, если всё пошло не так?
Если хочешь откатить изменения назад
Если ты применил миграцию, но понял, что она ошибочна:
alembic downgrade -1

(Это откатит базу на один шаг назад. После этого можно удалить плохой файл миграции и создать новый).
Если миграция не видит изменения ("No changes detected")

1. Проверь, импортирована ли твоя модель в файле alembic/env.py в переменную target_metadata.
2. Убедись, что ты сохранил файл models.py перед запуском команды.
Если база заблокирована или возник конфликт
Если ты случайно создал "разветвление" миграций (две миграции имеют одного родителя), используй:
alembic merge -m "merge_heads" <hash1> <hash2>

🚀 Важное правило для CI/CD
Никогда не меняй файлы в папке alembic/versions/ вручную после того, как ты закоммитил их в Git и отправил в репозиторий. Если нужно что-то поправить — создавай новую миграцию.
Совет: В твоем случае с проектом на FastAPI, при запуске на сервере через Docker или напрямую, не забудь добавить команду alembic upgrade head в скрипт запуска (entrypoint), чтобы база обновлялась автоматически.
